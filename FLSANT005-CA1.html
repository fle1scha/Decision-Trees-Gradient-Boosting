After dividing the mealie data into training and test sets using an 80/20 split, a decision tree was fit to the data using all features.

set.seed(13) 
index <- sample(1:nrow(mealie), 0.8*nrow(mealie))
train <- mealie[index,]
test <- mealie[-index,]
tree.mealie <- tree(quality ~ ., data=train)

The feature space has been split into 16, with each split chosen because it causes the greatest reduction in the Gini index - the defaut measure of node purity. Additionally, the tree was not split further because a stopping criteria was reached - either the nodes were too small or the within-node deviance was not large enough. The training misclassification rate shows that the tree does not model the data perfectly, as 58 of the 1000 observations are incorrectly categorised. This is acceptable, as a perfect training data classifier would not predict new observations accurately. Lastly, only 4 of the 5 explanatory variables were used to split the training data. This means that at no point in the recursive binary splitting process did a split on height provide the largest redution in the Gini index.

As shown in ?tree.control, a tree that fits the data perfectly can be modelled by setting the minimum within-node deviance (mindev) to 0, and minimum node size to 2. By relaxing these parameters, the tree is able to continue splitting until all of the nodes are the homogenous.

set.seed(13) 
tree.mealie.relaxed <- tree(quality~ ., data=train, control=tree.control(nrow(train), minsize=2, mindev=0))

The misclassification rate of 0% shows that each node is correctly classified, and it can be inferred that each node is homogenous. Using the which.min function, R returns a sub-tree with 45 terminal nodes as the tree with the lowest cross-validation error.

set.seed(13)

cv.mealie$size[which.min(cv.mealie$dev)]

However, upon manual inspection of the cross-validations errors and sub-tree sizes, it is evident that a subtree of size 24 produces the same cross-validation error as the 45-node subtree. The reason the which.min functions returns 45 is because it finds the first occurence of a minimum value, and the tree size decreases from 45.

Although sub-trees with more terminal nodes produce lower misclassification error rates than a 24-node tree, I decided to prune the tree to 24 nodes to reduce model variance and increase interperability. Resultingly, the tree no longer perfectly models the training data, and so the bias of the model has increased. This reduces overfitting to some extent and, as a result, the pruned tree may be a better predictor on the test subset.

pr.mealie <- prune.misclass(tree.mealie.relaxed, best = 24)

The snip.tree function shows the nodes that are the roots of the sub-trees that are to be pruned. The tree is nearly half the size of the full 45-node tree, and has a misclassification rate of 2.3%. Once again, the height variable has not been used to split the feature space, as it does not provide the highest reduction in the Gini idex for any split in a 24-node tree.

To identify the optimal hyperparameters for a random forest model, namely the number of trees and the number of features considered at each split, I first investigated the OOB MSE of a 500-tree bagged model and a 500-tree random forest model. The plot below shows the decrease in OOB MSE as the number of trees for each model increases. Bagging is just a special case of a random forest where m = p, and the default m for the randomForest() function is \[\sqrt(p)\]. There are two main takeaways from the plot: firstly, the bagging OOB MSE is much lower than that of the random forest, and secondly, the test error approaches a minimum around 250 trees for both the bagging and random forest models. Random forests will not overfit if B increases, so it is sensible to use a value of B where the error rate has settled down. Furthermore, the differene between 250 trees and 500 trees is negligble, and the computational cost of 500 trees is not worth the decrease in test error in my opinion.

Suppose that there is one very strong predictor in the data set, along with a number of other moderately strong predictors. Then in the collection of bagged trees, most or all of the trees will use this strong predictor in the top split. Consequently, all of the bagged trees will look quite similar to each other. Hence the predictions from the bagged trees will be highly correlated. Unfortunately, averaging many highly correlated quantities does not lead to as large of a reduction in variance as averaging many uncorrelated quantities.
